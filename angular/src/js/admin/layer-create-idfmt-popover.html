Each feature added to the system must have an id that identifies it uniquely within its layer.

If this layer is updated in the future (e.g. boundaries re-defined) it may be desireable to allow the feature data to be updated in place rather than recreated. For example if other entities are to be associated with a given feature to prevent data integrity problems.

The property format syntax is basic sprintf format without surrounding " characters. Unlike standard sprintf you can wrap optional properties in [].

Example: If your features had a unique property, GEOID that is guaranteed to remain constant moving forward you could specify a format like %s,GEOID.

Example: If your features had two properties then when put together would guarantee uniqueness, and remain constant moving forward, you could specify a format like %s.%s,MAINID,SUBID.
